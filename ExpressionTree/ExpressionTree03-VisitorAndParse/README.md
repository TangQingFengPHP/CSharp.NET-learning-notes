# 03 Visitor 与解析

## 详解

表达式树真正进入实战，通常离不开 `ExpressionVisitor`。

它的价值：

- 遍历整棵树
- 查找特定节点
- 改写节点
- 为翻译器、优化器、规则引擎做准备

## 常见用途

- 把常量统一替换
- 把字段名翻译成数据库列名
- 生成 SQL / DSL
- 做表达式标准化

## 补充说明

如果表达式里是 `product.Price > 100m`，那么常量节点类型就是 `decimal`。

这时你用 `100` 去匹配，通常匹配不到；要用 `100m` 才能正确命中。

原因不是“值看起来一样就行”，而是表达式树里的 `ConstantExpression` 会严格保留常量的真实类型：

- `100` 是 `int`
- `100m` 是 `decimal`

在 Visitor 里做常量替换时，既要看值，也要看类型。
