# 04 项目实践

## 实践思路

这个示例模拟订单支付成功后的状态流转。

设计目标：

- `OrderPaidStateTransition` 使用 `sealed`，保证状态迁移规则不会被子类偷偷改写。
- 对外暴露 `IOrderStateTransition` 接口，测试和替换实现时依赖接口而不是继承具体类。
- 体现真实项目里常见的建议：`sealed` 与接口搭配使用，而不是裸奔。

## 关键结论

- 如果你需要扩展能力，优先开放接口。
- 如果你需要锁定某个实现，使用 `sealed class`。
- 如果你需要限定继承链中的某个成员不能再改，用 `sealed override`。
