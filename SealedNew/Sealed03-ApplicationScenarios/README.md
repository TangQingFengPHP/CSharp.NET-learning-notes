# 03 应用场景

## 常见场景

1. 值对象/规则对象
   - 语义稳定，不希望子类改掉等价性或校验逻辑。
2. 安全组件
   - 例如 Token、权限、签名器，不允许通过继承绕过核心限制。
3. SDK 最终实现类
   - 对外只暴露接口，内部实现类用 `sealed` 收口。
4. 模板方法的终态节点
   - 父类开放虚方法，中间层完成最终实现后用 `sealed override` 阻止再改。
5. 性能敏感的热点对象
   - 某些场景里，JIT 对 sealed 类型更容易做去虚拟化优化，但这不是首要理由。

## 不适合的场景

- 业务仍在频繁变化，需要继承扩展
- 测试替身依赖继承 mock，而不是接口 mock
- 作为框架扩展点的基类
