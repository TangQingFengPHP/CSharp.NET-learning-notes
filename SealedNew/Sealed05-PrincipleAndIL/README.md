# 05 原理解析

## 编译器层面

C# 编译器会把 `sealed` 编译进类型元数据或成员元数据中：

- `sealed class`：类型被标记为不可继承
- `sealed override`：重写成员被标记为最终实现（final）

## CLR/JIT 层面

这会带来两个直接结果：

1. 运行时知道这个继承链到此结束
2. 某些调用点更容易被优化为非虚调用或被内联

但请注意：

- 不是所有 `sealed` 都一定显著提速
- 性能收益依赖 JIT、调用上下文、是否跨程序集等条件
- 设计边界清晰始终比“赌一点可能的性能收益”更重要

## IL 视角的理解

你可以把它粗略理解为：

- `sealed class` => 类型声明上有 `sealed`
- `sealed override` => 方法声明上常见 `final virtual`

也就是“它仍然是 override 来的虚槽位成员，但到这一层为止不能继续 override”。
